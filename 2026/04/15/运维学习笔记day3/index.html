一、数据流重定向与管道

1.1 标准流

Linux 系统中每个进程启动时，都会自动打开 3 个标准文件描述符（数据流），这三个数据流是 Linux 进程输入输出的「标准通道」，通过重定向可以灵活控制数据的来源和去向，是 Linux 命令行的核心基础。

流名称 代码 说明
标准输入 (stdin) 0 程序读取数据的来源，默认是键盘
标准输出 (stdout) 1 程序正常输出的目的地，默认是终端屏幕
标准错误 (stderr) 2 程序错误或警告信息的输出地，默认也是终端屏幕

1.2 输出重定向

操作符 作用 示例
> 或 1> 覆盖写入正确输出 echo "hello" > file.txt
>> 追加写入正确输出 echo "world" >> file.txt
2> 覆盖写入错误输出 ls /notexist 2> error.log
2>> 追加写入错误输出 ls /notexist 2>> error.log
&> 或 >& 正确 + 错误一起覆盖写入 ls /tmp &> all.log
2>&1 将错误输出合并到正确输出流 ls /tmp > out.log 2>&1
/dev/null Linux 黑洞，丢弃所有写入的数据 ls /tmp > /dev/null 2>&1

1.3 输入重定向

操作符 作用 示例
< 从文件读取输入 cat < file.txt
<< 结束标记 Here Document，读取多行输入直到标记结束 cat > file << EOF

1.4 管道（|）

  • 将前一个命令的标准输出作为后一个命令的标准输入。
  • 管道默认不传递标准错误，若需传递错误流，需先使用 2>&1 合并。

1.5 tee 命令（分流器）

从标准输入读取数据，同时写入标准输出和指定文件，常用于调试管道中间结果。

1.6 后台运行

  • 命令末尾加 &：将命令放入后台运行（仅适用于非交互式程序）。
  • jobs：查看当前终端下的后台任务。
  • Ctrl + C：终止前台进程。

3.2 grep 过滤

grep 是强大的文本搜索工具，能在文件中查找包含指定字符串的行，可结合管道命令，具体命令为 grep [选项] "查找内容" 文件名。

选项 作用 示例
-n 显示行号 grep -n root /etc/passwd
-v 反向匹配（排除） grep -v nologin /etc/passwd
-i 忽略大小写 grep -i d /etc/passwd
-A 数字 显示匹配行及后 N 行 grep -A2 halt /etc/passwd
-B 数字 显示匹配行及前 N 行 grep -B2 sshd /etc/passwd
-C 数字 显示匹配行及前后 N 行 grep -C2 ftp /etc/passwd
-c 统计匹配行数 grep -c nologin /etc/passwd

正则表达式示例：

表达式 含义 示例
^root 以 root 开头 grep "^root" /etc/passwd
sh$ 以 sh 结尾 grep "sh$" /etc/passwd
^[0-9] 以数字开头 grep "^[0-9]" file
^$ 空行 grep -v "^$" file（显示非空行）

find 是强大的实时文件查找工具。

基本格式：

常用查找条件：

条件 说明 示例
-name 按文件名（支持通配符） find /etc -name "host*"
-type f/d 按类型（文件/目录） find / -type f -name "*.conf"

逻辑组合：
